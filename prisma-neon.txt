


Flash FC: Backend Documentation (Beginner Edition)
1. The Database (Neon)
* What it is: A "Cloud" home for your data. Think of it as a super-powered Excel sheet that lives on the internet 24/7.
* Why we use it: Instead of saving data on your MacBook (which disappears when you turn it off), Neon keeps your players and officials safe and accessible from anywhere in the world.
* Key Concept: We used PostgreSQL, which is the "language" this database speaks.
2. The Translator (Prisma)
* What it is: An ORM (Object-Relational Mapper).
* Why we use it: Databases speak SQL (which is hard). JavaScript speaks Objects (which is easy). Prisma sits in the middle and translates your JavaScript code into Database commands automatically.
* The Schema (schema.prisma): This is your Blueprint. If it’s not in the schema, the database won’t allow it. It protects your data from being messy.
3. The Connection Bridge (.env)
* What it is: A secret file containing your "keys" to the Neon house.
* Why we use it: You never put your database password in your main code (where hackers could see it). The .env file keeps your credentials private.
* DATABASE_URL vs DIRECT_URL: Next.js is fast and opens many connections; the DATABASE_URL uses a "Pooler" to manage them. The DIRECT_URL is a straight pipe used only when we are building or changing tables.
4. The Safety Lock (lib/prisma.ts)
* What it is: A "Singleton" pattern.
* Why we use it: Every time you save code in Next.js, the server restarts. Without this file, Next.js would try to open 100 new connections to Neon every minute, eventually crashing your database. This file says: "If a connection is already open, just use that one."

The Commands Cheat Sheet (What they actually do)
Command	What it does	Why we run it
npx prisma init	Creates the prisma folder.	To start the backend engine.
npx prisma migrate dev	Takes your schema.prisma and "pushes" it to Neon.	To physically create or update the tables in the cloud.
npx prisma generate	Creates a local map of your database for your code.	So when you type prisma.user., your Mac knows exactly what fields exist (Auto-complete).
npx prisma studio	Opens a local "Admin Panel" in your browser.	So you can see, edit, and delete data without writing any code.
